Servicios Web de DynHost

DynHost proporciona servicios web para poder realizar tareas automatizadas desde otros programas automatizados. Estas tareas, se definen a continuación:

  • Actualización DynHost
  • Dirección IP
  • Actualización de clave de email
  • Actualización de clave de FTP

Volver

Actualización de DynHost

El sistema de actualización se ha basado en el protocolo propuesto por la empresa DynDNS® de actualización automática, que está implementado en clientes para Windows, Linux y MacOS, así como en algunos routers y/o modems de acceso ADSL. El sistema se basa en el protocolo HTTP (RFC 2616).

Una prueba que puede emplearse es:

http://usuario:clave@dynhost.es/nic/update/?hostname=mihost

El usuario y la clave pueden ser introducidos vía diálogo de autenticación en caso de querer actualizar por navegador web. La forma más sencilla de actualizar la IP, en sistemas basados en Unix (tipo Linux, BSD y MacOS), sería configurando un proceso cron que cada poco tiempo hiciera la petición.

Los parámetros que se pueden pasar a la URL son los siguientes:

Parámetro Descripción
hostname El hostname es el nombre del dominio (sin .dynhost.es) que se ha dado de alta. Es un valor obligatorio.
myip Este valor es opcional, ya que si no se indica, se autodetecta y toma de la propia conexión.

Dirección IP

Este servicio web proporciona la dirección IP desde la que se realiza la consulta. Se puede probar desde el propio navegador a través de la siguiente URL:

http://dynhost.es/nic/checkip/

Volver

Actualización de clave de email

DynHost proporciona buzones de email que pueden configurarse dentro de los dominios reales (en este caso no servirían para dominios DynHost), y a través de este servicio web, la clave puede ser modificada.

Este servicio web está pensado para poder implementarse como plugin o similar dentro de un sistema webmail o similar, incluso a través de una web de gestión propia de la empresa, con lo que puedan unificarse las claves que se dan en los accesos de los equipos (o sistemas CRM o ERP) y la que se tiene en el webmail.

Los parámetros que se pueden pasar a la URL son los siguientes:

Parámetro Descripción
username El nombre del usuario al que se le cambiará la clave (sin dominio).
newpass La nueva clave que se asignará al usuario.

Un ejemplo:

http://usuario:clave@dynhost.es/nic/mail/password/?username=miusuario&newpass=123456

Volver

Actualización de clave de FTP

DynHost proporciona a los usuarios que disponen de espacio de hosting, la posibilidad de acceder a este espacio a través de cuentas FTP. Las cuentas pueden crearse desde el panel de administración, pero si se desea integrar el cambio de clave de un usuario específico en otra interfaz o programa, este servicio web le permite realizar esa tarea.

Los parámetros que se pueden pasar a la URL son los siguientes:

Parámetro Descripción
username El nombre del usuario al que se le cambiará la clave.
newpass La nueva clave que se asignará al usuario.

Un ejemplo:

http://usuario:clave@dynhost.es/nic/ftp/password/?username=miusuario&newpass=123456
